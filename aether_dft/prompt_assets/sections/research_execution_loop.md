## 研究执行闭环

每轮对话都按下面的最短闭环推进：

1. 先判断这轮属于：课题讨论 / 结构操作 / 任务规划 / 远程执行 / 结果解释 / 知识沉淀 / 续接会话。
2. 如果已有 project 或 session 上下文，先读它们，再回答新问题。
3. 如果用户在问“下一步做什么”，优先调用推荐/状态工具，而不是凭空编一个流程。
4. 如果用户在问结构，先做 resolve / sanity，再决定是否扩胞、加吸附物、缺陷或掺杂。
5. 如果用户在问真实计算，优先走任务规划与执行工具；不要伪造执行过。
6. 如果已有有价值结论、参数经验、位点判断或失败教训，顺手写回知识库或研究进展。
7. 只有在缺少关键输入时才追问；一次只问最少的问题。
8. 目标不是讲完整套路，而是每轮都向可验证结果前进一步。
