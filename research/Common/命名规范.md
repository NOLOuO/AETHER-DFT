# 命名规范

> AI 必读。所有新建 .md / PDF / 文件夹必须遵守本规范。
> 命名是 AI 找到笔记的唯一线索——**命名好,等于建好了 AI 的索引**。

---

## 核心原则

1. **能从文件名一眼看出内容**,不需要打开
2. **同主题文件名字段一致**,便于 grep / glob
3. **日期写在前面**,便于按时间排序
4. **关键词写在中间**,便于检索
5. **不用空格**,用 `-` 或 `_`

---

## 各文件夹命名约定

### 研究计划/

```
YYYY-MM_主题_可选关键词.md
```

**示例**:
- `2026-05_F中心计算方案.md`
- `2026-04_MCH-TS1-TS6路径方案.md`
- `2026-05_Ru团簇构建-界面模型.md`

**要点**:日期精确到月即可,主题写"这份方案在解决什么问题"。

---

### 文献/

```
作者年份_期刊缩写_短标题.pdf
```

**示例**:
- `Wang2024_JACS_F-center-MSR-Ru-Al2O3.pdf`
- `Smith2023_Nature_PtBr-MCH-dehydrogenation.pdf`

**要点**:
- 作者用第一作者姓氏(不加名)
- 年份四位数字
- 期刊用通用缩写(JACS / Nature / ACS Catal / JCP)
- 短标题英文,空格用 `-`

---

### 文献参考/

```
作者年份_期刊缩写_短标题.md
```

**和对应 PDF 同名,只改后缀**。这样 AI 一看 .md 就知道对应哪篇 PDF,反之亦然。

**示例**:
- `Wang2024_JACS_F-center-MSR-Ru-Al2O3.md`(精读笔记)
- `Wang2024_JACS_F-center-MSR-Ru-Al2O3.pdf`(原文)

---

### deep research/

```
YYYY-MM-DD_主题_来源.md
```

**示例**:
- `2026-05-18_MSR机理综述_Gemini.md`
- `2026-05-10_F中心研究现状_GPT-DeepResearch.md`

**要点**:日期精确到日,来源标注用哪个工具产出,便于回溯质量。

---

### Learning/

```
结论一句话浓缩.md
```

**最重要的命名场景**——Learning 文件名直接是结论本身的一句话。

**示例**:
- `F中心U值选择_PBE+U应用于O2p而非Ru4d.md`
- `MAGMOM初始化_triplet收敛需手动指定方向.md`
- `MACE-NEB初猜_FIRE比LBFGS稳定.md`
- `d带中心_4Br-Pt偏离来自几何效应.md`

**反面教材**(不要这样命名):
- ❌ `2026-05-15_笔记.md`(不知道讲什么)
- ❌ `Learning1.md`(完全无意义)
- ❌ `F中心.md`(太宽泛,以后还会有别的 F 中心 Learning)

**要点**:
- 文件名前半部分是**主题**,后半部分是**关键结论**,用 `_` 分隔
- grep 一个主题词(如 "MAGMOM")应该立刻命中所有相关 Learning

---

### 研究进展.md

固定文件名,每个项目根目录一个,**不要加日期**(因为它本身是日期序列)。

---

## 文件夹命名

- 中文 / 英文均可,但**项目顶级文件夹**用 `项目缩写-体系` 格式
- 例:`MSR-Ru-Al2O3`、`MCH-Pt-Br`
- 子文件夹用中文(语义最清楚,且不与 AI 工具冲突)

---

## 反规范快速检查

如果你/AI 想新建一个文件,问自己三个问题:

1. **三个月后我自己看到这个文件名,能不能猜出内容?**
2. **AI grep 一个关键词,能不能命中?**
3. **同类型已有文件,我的新文件名风格一致吗?**

三个都"是" → 命名合格。任一"否" → 重命名。
